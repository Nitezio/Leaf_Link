Severity-Ranked Findings
1. High - Cart state is not fully reactive or durable.
	- The cart sheet does not listen live to `AppState`, and cart quantity changes are not persisted immediately.
	- Impact: the UI can go stale while the sheet is open, and quantities can be lost after restart.

2. High - Authentication is still local-only.
	- Firebase Auth exists as a service, but the app UI still uses local session state.
	- Impact: there is no real backend sign-in, account recovery, or durable authenticated session.

3. Medium - Plant images can fail for local file paths.
	- `PlantCard` currently assumes network images, so gallery-picked local files may not render correctly there.

4. Medium - Community dialogs rely on analyzer suppression.
	- `community_tab.dart` currently suppresses `use_build_context_synchronously` at the file level.
	- Impact: the code passes analysis, but future async UI changes are easier to get wrong.

5. Medium - Firestore data parsing is brittle.
	- The sync layer assumes a strict document shape, so malformed data could throw at runtime.

6. Low - Several features remain UI prototypes.
	- Scan/AR, checkout, notifications/scheduler, and deeper backend sync are still not fully implemented.
